# 未来路线图

## 近期计划

- 将截图占位逻辑替换为真实但轻量的 OCR / HUD 提取原型
- 增加本地 SQLite 历史记录
- 支持对两个可能动作进行并排比较
- 在前端增加更清晰的置信度解释区域

## 中期计划

- 引入多游戏知识包机制
- 支持更长的回合历史输入
- 为贡献者增加分析 trace，方便调试建议质量
- 在本地 Markdown 策略文档之上增加轻量 RAG

## 长期方向

- 支持第二个可公开演示的游戏场景
- 支持用户自定义知识包
- 支持更真实的截图理解
- 支持更完整的训练型复盘体验

## 明确不做的方向

以下方向不属于本项目范围：

- 游戏自动化
- 作弊脚本
- 输入注入
- 内存读取
- 反作弊规避
- 自动对战 Bot

GameBuddy Agent 的长期方向始终是：

- 教学
- 策略辅助
- 局势解释
- 赛后复盘

## 适合开源贡献者的 Good First Issues

- 新增一份示例战局并补充对应 Prompt
- 为前端 JSON 输入加更友好的格式校验提示
- 为策略代理增加更多边界场景测试
- 为复盘页增加复制总结按钮
- 增加第二套首页展示主题

## 更适合中级贡献者的 Issues

- 接入 SQLite 做本地历史记录
- 设计多游戏知识包加载机制
- 为截图模式接入 OCR 原型
- 增加“动作 A / 动作 B”对比接口与前端页面
